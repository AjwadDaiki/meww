## Stratégie

### Langues V1 (launch)

1. **Français** (FR) — marché natif, Ajwad FR-native, TikTok FR très actif

4. **Allemand** (DE) — marché premium EU, haut pouvoir d'achat
5. **Italien** (IT) — marché EU, love for cats (meme tier)
6. **Portugais** (PT-BR prioritaire) — Brésil = TikTok deuxième pays au monde

### Jamais

- Pas de chinois simplifié V1 (régulation Stripe + Replicate)
- Pas de russe (contraintes paiement)
- Pas d'arabe V1 (RTL trop coûteux à bien faire)

## Stack technique

### Librairie : `next-intl`

**Pourquoi :**
- Standard Next.js 15 App Router
- Type-safe (detects missing keys)
- SSR-compatible
- Routing `/fr/`, `/en/`, `/es/` natif
- Gère pluralisation, dates, currencies

### Installation

```bash
pnpm add next-intl
```

### Config

```typescript
// apps/web/next.config.ts
import createNextIntlPlugin from 'next-intl/plugin';
const withNextIntl = createNextIntlPlugin('./src/i18n/request.ts');

export default withNextIntl({
  // ... Next config
});
```

```typescript
// apps/web/src/i18n/config.ts
export const locales = ['fr', 'en', 'es'] as const;
export const defaultLocale = 'fr';

export type Locale = typeof locales[number];
```

### Règle pour les traductions

- **Le FR est la référence** (Ajwad FR-native, contrôle qualité direct)
- **EN traduit par native speaker** (pas juste Google Translate — le ton cinéma se perd)
- **ES idem**, avec sensibilité Amérique Latine vs Espagne (neutral LATAM de préférence)
- **Le vocabulaire cinéma** doit être adapté culturellement, pas juste traduit :
  - FR : "screen test" → "screen test" (anglicisme OK)
  - EN : "screen test" → "screen test"
  - ES : "screen test" → "casting"
  - DE : "screen test" → "Probeaufnahme"

## Routing

```
meowreel.com/         → redirect to detected locale (ex: /fr)
meowreel.com/fr       → FR landing
meowreel.com/en       → EN landing
meowreel.com/es       → ES landing
meowreel.com/fr/scene/midnight-porch-musician
meowreel.com/en/scene/midnight-porch-musician
...
```

### Détection auto

```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';
import { locales, defaultLocale } from './src/i18n/config';

export default createMiddleware({
  locales,
  defaultLocale,
  localeDetection: true,  // utilise Accept-Language header
});

export const config = {
  matcher: ['/((?!api|_next|favicon.ico|fonts|.well-known).*)'],
};
```

### Switcher de langue

Placé dans le header :

```tsx
<LocaleSwitcher>
  <Option value="fr">FR</Option>
  <Option value="en">EN</Option>
  <Option value="es">ES</Option>
</LocaleSwitcher>
```

Persist le choix utilisateur dans cookie `NEXT_LOCALE`.

## SEO multi-langue

### Hreflang tags

```html
<link rel="alternate" hreflang="fr" href="https://meowreel.com/fr" />
<link rel="alternate" hreflang="en" href="https://meowreel.com/en" />
<link rel="alternate" hreflang="es" href="https://meowreel.com/es" />
<link rel="alternate" hreflang="x-default" href="https://meowreel.com/en" />
```

### Sitemap par langue

```xml
<url>
  <loc>https://meowreel.com/fr/scene/midnight-porch-musician</loc>
  <xhtml:link rel="alternate" hreflang="en" href="https://meowreel.com/en/scene/midnight-porch-musician" />
  <xhtml:link rel="alternate" hreflang="es" href="https://meowreel.com/es/scene/midnight-porch-musician" />
</url>
```

### Meta par langue

Chaque page a ses meta tags traduits via next-intl :

```tsx
// apps/web/app/[locale]/page.tsx
import { getTranslations } from 'next-intl/server';

export async function generateMetadata({ params }: { params: { locale: string } }) {
  const t = await getTranslations({ locale: params.locale, namespace: 'seo.home' });
  return {
    title: t('title'),
    description: t('description'),
  };
}
```

## Stripe multi-langue

Stripe Checkout supporte 30+ langues. On passe la locale du user à la session :

```typescript
const session = await stripe.checkout.sessions.create({
  // ...
  locale: userLocale,  // 'fr', 'en', 'es', etc.
});
```

Stripe traduit tout le checkout (labels, Apple Pay prompt, etc.) automatiquement.

## Emails multi-langue

MailerLite permet de templater en plusieurs langues. On a 3 templates :
- `delivery_email_fr`
- `delivery_email_en`
- `delivery_email_es`

Le worker choisit le bon template selon `order.locale`.

## Catégories : noms localisés

Chaque catégorie dans MongoDB a 3 clés :

```typescript
{
  slug: 'midnight-porch-musician',
  name: {
    fr: 'Musicien du Perron Nocturne',
    en: 'Midnight Porch Musician',
    es: 'El Músico de la Terraza a Medianoche'
  },
  tagline: {
    fr: 'Le trend Sora 2 qui cartonne',
    en: 'The viral Sora 2 trend',
    es: 'El trend viral de Sora 2'
  }
}
```

L'API renvoie automatiquement le nom dans la bonne langue selon `Accept-Language` ou route locale.
